## MQ+ транспорт
Два канала связи: RabbitMQ + MQTT (в зависимости от направления и контекста).
###### Ресиверы (обработчики команд) регистрируются в соответствующих RabbitMQ очередях, а ответ отправляют (возвращают вызывающему, когда это предусмотрено) через MQTT.
Любой микро-сервис может иметь "точки доступа" для "выполнения команд извне".
К таким "точкам доступа" может обратиться любой "внешний потребитель" с целью "выполнения какой-то команды".
Для каждой такой потенциальной команды создаётся + регистрируется персональный ресивер, который в свою очередь создаёт очередь в RabbitMQ и тут же приступает к `Listen` этой очереди на протяжении всего срока жизни службы.

"Внешние клиенты" (потребители) могут обращаться к таким "точкам доступа" для "выполнения команды". Если "потребитель" хочет получить результат выполнения своей команды, тогда он предварительно определяет уникальное/одноразовое имя MQTT топика и подписывается на этот топик на период ожидания ответа/результата выполнения команды.
Отправляя команду через RabbitMQ клиент сообщает и имя MQTT топика на который клиент уже подписался и ждёт ответ.
В процессе обработки команды микро-сервис возвращает результат в MQTT топик, если он не объявлен.
В тех случаях, когда ответ не требуется: потребитель (вызывающий клиент) не определяет имя MQTT топика для ответа и не ожидают этого ответа по принципу "отправил и забыл".
